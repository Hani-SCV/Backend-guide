### 개념


> - HTTP 통신에 SSL/TLS 암호화 계층을 더한 프로토콜  
>   
> - 클라이언트와 서버 간 데이터의 기밀성, 무결성, 인증 제공  
>   
> - 중간자 공격, 도청, 데이터 변조 방지 목적


### 주요 구성요소

- **SSL/TLS**: 암호화, 인증, 무결성 보장
- **인증서(X.509)**: 서버 신원 확인 및 공개키 전달
- **대칭키 암호화**: 데이터 암호화에 사용
- **비대칭키 암호화**: 키 교환과 인증에 사용

### 동작 흐름 (핸드셰이크)

1. 클라이언트가 서버에 연결 요청 (ClientHello)
2. 서버가 인증서와 공개키 전달 (ServerHello)
3. 클라이언트가 세션 키 생성 후 서버 공개키로 암호화해 전송
4. 서버가 복호화 후 세션 키 공유
5. 이후 대칭키 암호화로 안전한 통신 시작

### 장점

- 통신 내용 암호화로 데이터 유출 방지
- 서버 신원 검증으로 피싱·위조 방지
- 데이터 무결성 검증으로 변조 탐지

### 주의사항

- 인증서 신뢰 체인과 유효성 검증 필수
- 만료된 인증서 사용 금지
- HTTPS 강제 적용 및 HSTS 설정 권장
- 중간자 공격 예방을 위한 TLS 최신 버전 사용

### Tip

- Let’s Encrypt 등 무료 인증서 활용 가능
- 서버와 클라이언트 모두 최신 TLS 버전 지원 여부 점검
- HTTP 요청을 HTTPS로 리다이렉트 설정
- TLS 세션 재사용과 세션 티켓 설정으로 성능 최적화

### 개념


> - SSL(Secure Sockets Layer)은 HTTPS 암호화의 초기 프로토콜  
>   
> - TLS(Transport Layer Security)는 SSL의 후속 버전으로 더 안전하고 발전된 프로토콜  
>   
> - 현재는 TLS가 사실상 표준이며, SSL은 폐기 상태


### 주요 기능

- 데이터 암호화 (기밀성 보장)
- 서버 및 클라이언트 인증
- 데이터 무결성 검증
- 세션 키 교환

### TLS 버전

- TLS 1.0, 1.1: 구버전, 보안 취약점 존재
- TLS 1.2: 널리 사용되는 안정적 버전
- TLS 1.3: 최신 버전, 성능과 보안 향상

### 동작 방식

- 핸드셰이크 과정에서 암호화 알고리즘, 키 교환 방식 협상
- 공개키 기반 암호화로 세션 키 교환
- 이후 대칭키 암호화로 데이터 송수신

### 주의사항

- SSL 2.0, 3.0 사용 금지 (취약점 다수)
- TLS 1.0, 1.1 지원 중단 권장
- 최신 TLS 1.2 이상 사용 필수
- 취약한 암호화 알고리즘 및 프로토콜 비활성화

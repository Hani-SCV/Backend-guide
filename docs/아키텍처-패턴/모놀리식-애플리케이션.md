### 개념


> - **하나의 단일 애플리케이션**으로 모든 기능이 통합된 구조  
>   
> - 모든 코드(비즈니스 로직, UI, 데이터 접근 등)가 한 프로세스 내에 존재


### 특징

- **단순한 배포**: 하나의 배포 단위로 관리
- **개발 초기 빠름**: 구조가 단순해 빠르게 개발 가능
- **직관적인 설계**: 전체 애플리케이션이 한 프로젝트 내에 존재

### 장점

- 개발, 테스트, 배포가 간편
- 모듈 간 호출이 로컬 함수 호출 수준으로 빠름
- 인프라 요구사항이 적음

### 단점

- 규모가 커질수록 유지보수 어려움
- 기능 추가 시 전체 재배포 필요
- 한 부분 문제 발생 시 전체 서비스 영향 가능성
- 확장성과 유연성 제한적

### 적합한 상황

- 소규모 프로젝트 또는 MVP
- 빠른 프로토타입 개발 필요할 때
- 팀 규모가 작고 복잡도가 낮을 때

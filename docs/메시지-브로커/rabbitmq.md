### 개념


> - 오픈소스 **메시지 브로커**  
>   
> - AMQP(Advanced Message Queuing Protocol) 기반으로 신뢰성 있는 메시지 큐잉 제공  
>   
> - 시스템 간 비동기 통신, 분산 처리에 사용


### 특징

- 다양한 메시지 패턴 지원 (큐잉, 퍼블리시/서브스크라이브, 라우팅 등)
- 메시지 확인, 재전송, 우선순위, TTL 설정 가능
- 플러그인으로 기능 확장 가능
- 관리용 웹 UI 제공

### 장점

- 안정적인 메시지 전달 보장
- 다양한 프로토콜과 언어 지원
- 유연한 라우팅과 확장성
- 활성화된 커뮤니티와 문서

### 단점

- 고가용성 설정과 클러스터링 복잡
- 높은 메시지 처리량에서는 성능 병목 가능
- 운영과 관리에 일정 수준의 학습 필요

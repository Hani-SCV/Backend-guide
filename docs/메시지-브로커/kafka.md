### 개념


> - 분산형 **스트리밍 플랫폼 및 메시지 브로커**  
>   
> - 대용량 실시간 데이터 파이프라인 구축에 최적화  
>   
> - 퍼블리시-서브스크라이브 모델 기반


### 특징

- 높은 처리량과 내결함성
- 데이터는 토픽(Topic)에 저장, 파티션으로 분산 관리
- 영구 저장(로그 저장소) 지원
- 소비자 그룹을 통한 병렬 처리 가능
- 스트림 처리 라이브러리(kafka-streams) 제공

### 장점

- 대규모 실시간 데이터 처리에 강점
- 높은 내구성과 확장성
- 다양한 클라이언트 언어 지원
- 복잡한 이벤트 처리 및 분석 가능

### 단점

- 초기 설정과 운영 복잡
- 학습 곡선이 높음
- 작은 메시지 처리 시 오버헤드 존재

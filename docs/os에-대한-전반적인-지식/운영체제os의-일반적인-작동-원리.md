## 운영체제(OS, Operating System)


> 💡 운영체제(OS) 는 컴퓨터 하드웨어와 소프트웨어 리소스를 관리하고, 사용자와 컴퓨터 간의 상호작용을 가능하게 하는 소프트웨어입니다. OS의 작동 원리는 여러 구성 요소와 개념으로 설명될 수 있습니다.


---


### 1. **프로세스 관리**

- **프로세스**: 실행 중인 프로그램. OS는 여러 프로세스를 관리.
- **스케줄링**: CPU, 메모리, I/O 자원을 효율적으로 할당하여 여러 프로세스가 동시에 실행되도록 함.

### 프로세스 상태

- **새로 생성(Ready)**: 프로세스가 실행될 준비가 되었으나 CPU를 기다리는 상태
- **실행 중(Running)**: CPU에서 실행되고 있는 상태
- **대기 중(Waiting)**: I/O 작업 등으로 대기 중인 상태
- **종료(Terminated)**: 실행을 완료한 상태

### 주요 스케줄링 알고리즘

- **FCFS (First-Come, First-Served)**: 도착 순서대로 처리
- **SJF (Shortest Job First)**: 가장 짧은 작업부터 처리
- **RR (Round Robin)**: 각 프로세스에 일정 시간씩 CPU를 할당하여 순환 처리

---


### 2. **메모리 관리**

- **주 기억 장치**: OS는 RAM을 관리하여 프로세스의 메모리 공간을 할당하고 해제
- **가상 메모리**: 디스크의 일부를 메모리처럼 사용하여 물리적 메모리 부족 시 대체

### 메모리 관리 기법

- **페이지 교체**: 메모리 페이지를 효율적으로 교체
- **페이징**: 메모리를 일정 크기 블록으로 나누어 관리
- **세그멘테이션**: 프로세스를 논리적 단위로 나누어 관리

### 가상 메모리 장점

- **프로세스 분리**: 각 프로세스가 독립적인 가상 메모리 공간을 가짐
- **메모리 확장**: 더 많은 프로그램 실행 가능
- **보안성**: 프로세스 간 메모리 접근 제한
- **효율적인 메모리 사용**: 자주 사용되지 않는 데이터를 디스크에 저장, 필요한 데이터만 RAM으로 불러옴

---


### 3. **파일 시스템 관리**

- **파일 및 디렉토리**: 파일 생성, 삭제, 읽기 및 쓰기 지원
- **권한 관리**: 파일과 디렉토리에 대한 접근 권한 설정

### 파일 시스템 종류

- **FAT (File Allocation Table)**: 간단하고 빠른 파일 시스템
- **NTFS (New Technology File System)**: 윈도우에서 사용
- **ext4**: 리눅스에서 널리 사용

---


### 4. **입출력(I/O) 관리**

- **장치 드라이버**: 하드웨어 장치와 상호작용하기 위한 소프트웨어
- **I/O 버퍼링**: 데이터 전송 원활화를 위해 I/O 작업을 버퍼링

### I/O 관리 기법

- **비동기 I/O**: I/O 작업 완료까지 기다리지 않고 다른 작업 수행
- **블로킹 I/O**: I/O 작업 완료될 때까지 프로세스 대기

---


### 5. **사용자 인터페이스**

- **명령 줄 인터페이스(CLI)**: 명령어로 OS와 상호작용
- **그래픽 사용자 인터페이스(GUI)**: 아이콘, 메뉴를 통해 사용자 친화적 인터페이스 제공

### UI 디자인 요소

- **아이콘**: 시각적인 그래픽 요소
- **메뉴**: 기능을 그룹화하여 사용자 접근 용이

---


### 6. **보안 및 접근 제어**

- **사용자 인증**: 사용자 이름과 비밀번호로 시스템 접근 제어
- **권한 제어**: 리소스 접근을 인증된 사용자만 가능하게 제어

### 보안 기법

- **암호화**: 데이터 보호를 위한 정보 변환
- **방화벽**: 네트워크 접근 제한

---


### 7. **네트워크 관리**

- **네트워크 연결**: OS는 다른 컴퓨터와 상호작용을 지원
- **데이터 송수신**: 네트워크 프로토콜 관리하여 데이터 전송

### 네트워크 프로토콜

- **TCP/IP**: 기본적인 인터넷 프로토콜
- **HTTP/HTTPS**: 웹 페이지 전송을 위한 프로토콜

---


### 8. **리소스 관리**

- OS는 시스템 리소스를 효율적으로 관리하여 프로세스가 필요한 리소스를 적시에 제공하고 충돌을 방지

### 리소스 관리 전략

- **우선순위 기반 할당**: 중요한 프로세스에 리소스 우선 할당
- **자원 예약**: 장기적으로 필요한 자원 미리 예약하여 효율성 높임

---


> 💡 커널(Kernel)은 운영체제의 핵심 구성 요소로, 하드웨어와 소프트웨어 간의 상호작용을 관리합니다.   
> - 커널 모드: 하드웨어 자원에 대한 완전한 접근 권한을 가지고 시스템의 중요한 작업 수행    
> - 사용자 모드: 제한된 접근 권한을 가지고 애플리케이션 실행

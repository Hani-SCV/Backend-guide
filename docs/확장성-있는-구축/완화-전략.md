### 개념


> - **완화 전략(Mitigation Strategy)**: 시스템 장애, 성능 저하, 보안 위협 등 **리스크 발생 시 영향 최소화**를 위한 사전/사후 대응 계획  
>   
> - 목표: 서비스 안정성 확보, 다운타임 최소화, 데이터 손실 방지


### 주요 전략

1. **장애 대비(Redundancy & Failover)**
    - 서버/서비스 중복 구성 → 장애 발생 시 자동 전환
    - 예: 로드 밸런서 + 다중 서버, DB 복제
2. **트래픽 관리(Traffic Shaping)**
    - 급격한 트래픽 증가 대응 → 요청 제한, 큐잉, Rate Limiting
    - 예: API 요청 제한, 버퍼링
3. **데이터 보호(Data Backup & Recovery)**
    - 정기 백업 + 복구 계획 → 데이터 손실 최소화
    - 예: DB 스냅샷, 클라우드 백업
4. **모니터링 & 알람(Monitoring & Alerting)**
    - 이상 징후 조기 탐지 → 신속 대응
    - 예: CPU/메모리 사용량, 에러율, SLA 초과 시 알람
5. **점진적 배포(Gradual Deployment)**
    - 블루-그린, 카나리 배포 활용 → 문제 발생 시 롤백 용이

### 장점

- 서비스 가용성 및 안정성 확보
- 장애 발생 시 **영향 범위 최소화**
- 운영 부담 감소 및 고객 신뢰 확보

### 단점 / 주의점

- 전략 구현 비용/복잡도 증가
- 과도한 중복 구성 → 리소스 낭비 가능
- 테스트 및 유지보수 필요

### 구현 예시


### 장애 대비 (로드 밸런서 + 다중 서버)


```plain text
upstream app_servers {
    server 192.168.0.101:3000;
    server 192.168.0.102:3000;
}

server {
    listen 80;
    location / {
        proxy_pass http://app_servers;
    }
}
```


### 트래픽 관리 (Rate Limiting - Nginx)


```plain text
http {
    limit_req_zone $binary_remote_addr zone=one:10m rate=10r/s;
    server {
        location /api/ {
            limit_req zone=one burst=20 nodelay;
        }
    }
}
```


### 데이터 백업 (PostgreSQL 예시)


```bash
pg_dump -U user dbname > backup.sql
# 복구
psql -U user dbname < backup.sql
```

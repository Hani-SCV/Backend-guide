### 🔍 개념

- **차이 이해하기(Differences Understanding)**: 유사 기술/패턴 간 **특징, 장단점, 사용 사례**를 비교하여 적절히 선택할 수 있는 능력
- 목표: 설계·구현 시 **최적의 기술 선택**, 문제 발생 시 **빠른 판단**

---


### ⚙️ 비교 방법

1. **기능(Functionality)** ⚙️
    - 어떤 기능을 제공하는지, 어떤 문제를 해결할 수 있는지
2. **성능(Performance)** ⚡
    - 처리 속도, 지연(latency), 확장성
3. **구현 난이도(Complexity)** 🛠
    - 개발/운영 난이도, 유지보수 용이성
4. **비용(Cost)** 💰
    - 하드웨어, 라이선스, 운영 비용
5. **사용 사례(Use Case)** 💡
    - 실제 서비스 환경에서 적합한 상황

---


### ✅ 장점

- 기술 선택 실수 최소화 ✅
- 설계·아키텍처 결정 속도 향상 ⚡
- 서비스 요구사항에 맞는 최적화 가능 🎯

---


### ⚠️ 단점 / 주의점

- 비교 항목이 많으면 분석 복잡도 증가
- 최신 기술/환경 변화 반영 필요
- 경험 부족 시 직관에 의존할 수 있음

---


### 🛠 예시 비교


| 항목      | WebSocket  | SSE       | Long Polling |
| ------- | ---------- | --------- | ------------ |
| 통신 방식   | 양방향        | 단방향       | 단방향          |
| 연결 유지   | 지속         | 지속        | 요청마다 새 연결    |
| 브라우저 지원 | 대부분        | 대부분       | 모두           |
| 서버 부하   | 낮음         | 중간        | 높음           |
| 활용 사례   | 채팅, 실시간 알림 | 뉴스, 주식 알림 | 구식 실시간 서비스   |


---


### 💡 활용 사례

- 실시간 통신 기술 선택 💬
- 캐싱 전략 선택 (Redis vs Memcached) ⚡
- 데이터베이스 선택 (SQL vs NoSQL) 💾
- 클라우드 아키텍처 결정 ☁️

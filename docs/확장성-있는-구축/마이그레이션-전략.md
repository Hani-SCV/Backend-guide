### 🔍 개념

- 마이그레이션(Migration): 기존 시스템/데이터를 **새 환경 또는 아키텍처로 이전**하는 과정
- 목표: **서비스 중단 최소화, 안정적 이전, 확장성 유지**

---


### ⚙️ 마이그레이션 전략 유형

1. **단계적(Migration by Phase)** 🐾
    - 서비스/모듈 단위로 이전
    - 장점: 리스크 최소화
    - 단점: 전체 완료까지 시간 소요
2. **블루-그린(Blue-Green Deployment)** 🔵🟢
    - 기존(Blue) + 새 환경(Green) 동시 운영
    - 전환 시 다운타임 최소화
    - 문제가 생기면 즉시 롤백 가능
3. **카나리(Canary Release)** 🐤
    - 일부 사용자에게 먼저 배포 → 문제 없으면 전체 롤아웃
    - 점진적 안정 검증 가능
4. **데이터 마이그레이션** 💾
    - **배치(Migration Batch)**: 점진적 데이터 이전
    - **실시간 동기화**: 서비스 중단 최소화

---


### ✅ 고려 사항

- 데이터 일관성 유지 (ACID, CAP 이론 고려)
- 트래픽 분산 및 로드 밸런싱 계획
- 롤백 계획 및 모니터링 체계
- 테스트 환경에서 충분히 검증 후 실제 적용

---


### 🛠 구현 예시


### 블루-그린 배포 (개념)


```plain text
[Blue 서버] ← 현재 운영
[Green 서버] ← 새 버전 배포
Switch Traffic → Green
문제 발생 → Rollback → Blue
```


### 카나리 배포 (개념)


```plain text
10% 사용자 → 새 버전 제공
모니터링 후 문제 없으면 100% 롤아웃
문제 발생 → 해당 사용자만 롤백
```


### 수평 확장 + 로드 밸런서 (Nginx 예시)


```plain text
upstream app_servers {
    server 192.168.0.101:3000;
    server 192.168.0.102:3000;
}

server {
    listen 80;
    location / {
        proxy_pass http://app_servers;
    }
}
```

### 💡 활용 사례

- 레거시 시스템 → 클라우드 마이그레이션 ☁️
- 트래픽 급증 대응을 위한 신규 인프라 이전 📈
- 무중단 배포 및 서비스 안정성 확보 🟢
- 모놀리식 → 마이크로서비스 전환 🔄

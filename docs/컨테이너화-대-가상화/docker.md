### 🧠 개념

- 컨테이너 기반 **애플리케이션 패키징 및 배포 플랫폼**
- 애플리케이션과 실행 환경을 하나의 이미지로 묶어 일관된 환경 제공
- 빠르고 가벼운 컨테이너 실행 및 관리 지원

---


### ⚙️ 주요 구성 요소

- **Docker Engine**: 컨테이너 생성, 실행, 관리
- **Docker Image**: 실행에 필요한 파일과 설정이 포함된 읽기 전용 템플릿
- **Docker Container**: 이미지를 기반으로 실행 중인 격리된 프로세스
- **Dockerfile**: 이미지를 빌드하기 위한 스크립트
- **Docker Hub**: 이미지 저장 및 공유를 위한 퍼블릭 저장소

---


### ✅ 장점

- 환경 일관성 보장 (개발→테스트→운영 동일 환경)
- 빠른 배포 및 스케일링
- 자원 효율적 (경량화된 실행 환경)
- 활발한 커뮤니티와 다양한 생태계

---


### ⚠️ 단점

- 상태 비저장 환경 (Stateless) 권장
- 가상화 수준보다 낮은 격리
- 네트워크 및 보안 설정 복잡도 존재

---


### 🛠 주요 기능

- 이미지 레이어 기반 효율적 저장 및 배포
- 볼륨을 통한 데이터 영속성 관리
- 네트워크 브리지, 호스트, 오버레이 지원
- Docker Compose로 다중 컨테이너 환경 정의 및 실행

---


### 활용 사례

- 마이크로서비스 배포
- CI/CD 파이프라인 자동화
- 로컬 개발 환경과 프로덕션 환경 일치화

### 🧠 개념

- **가상화 (Virtualization)**: 하드웨어 자원을 가상 머신(VM) 단위로 분할하여, 각각에 OS를 설치해 독립 실행 환경 제공
- **컨테이너화 (Containerization)**: OS 커널을 공유하면서 프로세스 단위로 격리하여 가볍고 빠른 실행 환경 제공

---


### 🛠 구조 비교


| 구분     | 가상화 (Virtual Machine)     | 컨테이너화 (Container)     |
| ------ | ------------------------- | --------------------- |
| OS 레벨  | 하이퍼바이저 위에 각 VM마다 별도 OS 구동 | 호스트 OS 커널 공유, 프로세스 격리 |
| 자원 사용  | 무거움, 각 VM에 OS 자원 필요       | 경량, OS 자원 공유          |
| 시작 시간  | 수십 초 이상                   | 수 초 이내                |
| 이동성    | VM 이미지 크고 무거움             | 이미지 작고 이식성 뛰어남        |
| 관리 복잡도 | VM 단위 관리, 상대적으로 무거움       | 컨테이너 단위 관리, 간편        |


---


### ✅ 장점

- **가상화**: 완전한 격리 환경, 다양한 OS 동시 운영 가능
- **컨테이너화**: 빠른 시작, 자원 효율성, 개발-운영 일관성

---


### ⚠️ 단점

- **가상화**: 무거운 리소스 사용, 느린 시작
- **컨테이너화**: 호스트 OS에 의존, 격리 수준 낮음

---


### 활용 사례

- 가상화: 다양한 OS 테스트, 완전한 분리 필요 환경
- 컨테이너화: 마이크로서비스, 빠른 배포, DevOps 환경

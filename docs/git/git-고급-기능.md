### **Git고급 기능의 필요성**


> Git의 기본 기능을 이해한 후, 고급 기능을 통해 더욱 효율적이고 유연하게 버전 관리를 수행할 수 있습니다. 이러한 기능들은 복잡한 프로젝트나 대규모 팀에서 특히 유용합니다.


### **1. Git Rebase**

- **정의**: `git rebase`는 한 브랜치의 변경 사항을 다른 브랜치에 적용하는 과정입니다. 이를 통해 커밋 기록을 더 깔끔하게 유지할 수 있습니다.
- **사용 예**:

    ```shell
    git checkout feature-branch
    git rebase main
    ```

- **장점**:
    - 병합 커밋을 방지하여 커밋 기록을 단순화합니다.
    - 깔끔한 프로젝트 이력을 유지할 수 있습니다.

### **2. Git Cherry-Pick**

- **정의**: 특정 커밋을 선택하여 다른 브랜치에 적용하는 기능입니다.
- **사용 예**:

    ```shell
    git checkout main
    git cherry-pick <커밋 해시>
    ```

- **장점**:
    - 원하는 커밋만 선택하여 필요한 변경 사항을 다른 브랜치에 손쉽게 적용할 수 있습니다.

### **3. Git Stash**

- **정의**: 작업 중인 변경 사항을 임시로 저장하고, 깨끗한 작업 환경으로 돌아가는 기능입니다.
- **사용 예**:

    ```shell
    git stash save "작업 중인 변경 사항"
    git stash pop
    ```

- **장점**:
    - 작업 도중 다른 작업을 해야 할 때 유용하며, 나중에 저장한 변경 사항을 쉽게 복원할 수 있습니다.

### **4. Git Hooks**

- **정의**: Git의 특정 이벤트(커밋, 푸시 등)에 대해 자동으로 실행되는 스크립트입니다.
- **사용 예**: `.git/hooks/` 디렉토리에 스크립트를 작성하여 사용합니다.
- **장점**:
    - 자동화된 작업을 통해 일관성 있는 프로세스를 유지할 수 있습니다. 예를 들어, 코드 커밋 시 자동으로 테스트를 실행할 수 있습니다.

### **5. Git Submodule**

- **정의**: 다른 Git 저장소를 하위 모듈로 포함시키는 기능입니다.
- **사용 예**:

    ```shell
    git submodule add <repository-url>
    ```

- **장점**:
    - 의존성이 있는 여러 프로젝트를 관리할 때 유용하며, 각 프로젝트의 버전을 독립적으로 관리할 수 있습니다.

### **6. Git Bisect**

- **정의**: 특정 버그를 찾기 위해 이진 검색 방법을 사용하는 기능입니다.
- **사용 예**:

    ```shell
    git bisect start
    git bisect bad   # 현재 커밋이 문제 있음
    git bisect good <좋은 커밋 해시>
    ```

- **장점**:
    - 문제가 발생한 커밋을 빠르게 찾아내어 디버깅 작업을 효율적으로 수행할 수 있습니다.

### **7. Git Worktree**

- **정의**: 동일한 저장소의 여러 브랜치를 동시에 체크아웃할 수 있게 해주는 기능입니다.
- **사용 예**:

    ```shell
    git worktree add <경로> <브랜치 이름>
    ```

- **장점**:
    - 여러 기능을 동시에 개발해야 할 때 각기 다른 디렉토리에서 작업할 수 있어 편리합니다.

### **8. Git Config**

- **정의**: Git의 다양한 설정을 관리하는 방법입니다.
- 개인 설정뿐만 아니라 팀이나 프로젝트별로 다양한 설정을 조정할 수 있습니다.
- **사용 예**:

    ```shell
    git config --global user.name "사용자 이름"
    git config --global user.email "이메일 주소"
    ```

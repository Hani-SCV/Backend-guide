### RDBMS (관계형 데이터베이스)

- 테이블 기반 정형 데이터 저장 방식
- SQL로 데이터 정의 및 조회
- ACID 트랜잭션 보장 → 데이터 정합성, 무결성 중점

특징

- 스키마 고정, 설계 구조 엄격
- 복잡한 조인, 정규화 설계 가능
- 예시: MySQL, PostgreSQL, Oracle

### NoSQL (비관계형 데이터베이스)

- 유연한 스키마로 다양한 데이터 형식 저장 가능
- 문서, 키-값, 컬럼, 그래프 등 다양한 모델 존재
- CAP 이론 기반 → 가용성(A), 파티션 내성(P) 중심 설계

특징

- 수평 확장성 우수 → 대규모 데이터에 유리
- 정규화보다 속도와 유연성 중시
- 예시: MongoDB, Cassandra, Redis

### 주요 비교


| 항목     | RDBMS            | NoSQL                |
| ------ | ---------------- | -------------------- |
| 데이터 구조 | 테이블 기반 (정형)      | 문서, 키-값 등 (비정형)      |
| 스키마    | 고정 스키마           | 유연한 스키마              |
| 트랜잭션   | ACID             | BASE                 |
| 확장 방식  | 수직 확장 (Scale-up) | 수평 확장 (Scale-out)    |
| 적합한 상황 | 복잡한 관계, 정합성 우선   | 대용량 처리, 유연한 구조 필요할 때 |


### 장단점 정리

- RDBMS
    - ✅ 정형 데이터, 관계 구조에 강함
    - ✅ 복잡한 쿼리 및 트랜잭션 처리에 유리
    - ❌ 스케일 아웃 한계, 초기 설계 중요
- NoSQL
    - ✅ 확장성, 성능, 비정형 데이터에 강점
    - ✅ 구조 유연 → 개발 속도 빠름
    - ❌ 복잡한 관계 표현 어려움, 정합성 보장 약함

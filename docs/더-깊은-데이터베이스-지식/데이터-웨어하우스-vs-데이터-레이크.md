### 데이터 웨어하우스 (Data Warehouse)

- 정형 데이터 저장·분석
- 목표: 신뢰성 높은 비즈니스 분석 데이터 제공
- 특징
    - ETL(추출·변환·적재)로 데이터 정제 후 저장
    - 저장 시 스키마 적용 (Schema-on-write)
    - OLAP, BI 시스템 중심
    - 예시: Snowflake, Amazon Redshift, Google BigQuery

### 데이터 레이크 (Data Lake)

- 정형·비정형 데이터 원시 저장
- 목표: 다양한 데이터 유연하게 저장·분석
- 특징
    - 데이터 원본 그대로 저장
    - 읽기 시 스키마 적용 (Schema-on-read)
    - 데이터 과학·머신러닝 워크로드에 최적
    - 예시: AWS S3 + EMR, Azure Data Lake, Hadoop HDFS

### 비교


| 구분     | 데이터 웨어하우스              | 데이터 레이크               |
| ------ | ---------------------- | --------------------- |
| 데이터 유형 | 정형                     | 정형 + 비정형              |
| 스키마 적용 | 저장 시 (Schema-on-write) | 읽기 시 (Schema-on-read) |
| 데이터 정제 | 사전 정제 필수               | 사전 정제 불필요             |
| 장점     | 정확성·신뢰성 높음             | 유연성·확장성 뛰어남           |
| 단점     | 비정형 처리 한계              | 데이터 품질·거버넌스 관리 어려움    |
| 주요 용도  | OLAP, 비즈니스 인텔리전스       | 데이터 과학, 빅데이터 분석       |


### 하이브리드 접근: 데이터 레이크하우스

- 웨어하우스의 관리·쿼리 성능과 레이크의 유연성 결합
- Delta Lake, Apache Iceberg, Databricks Lakehouse 등 솔루션

### Tip

- 목적에 따라 분리 운영: 자주 쓰는 분석은 웨어하우스, 실험적 데이터는 레이크
- 데이터 카탈로그·거버넌스 도구 도입 (Glue, Atlas 등)
- 중복 저장 최소화: 웨어하우스에 필요한 데이터만 ETL 로드
- 저장 비용 관리: 레이크는 저비용 스토리지, 웨어하우스는 컴퓨팅 효율 고려

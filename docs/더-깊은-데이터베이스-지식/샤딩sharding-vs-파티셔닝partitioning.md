### 샤딩 (Sharding)

- 데이터를 여러 서버(노드)에 나누어 저장하는 **분산 저장 기법**
- 주로 수평 확장(scale-out)을 통해 **처리량 증가 및 부하 분산**을 유도
- 각 샤드는 **독립된 데이터베이스 인스턴스**로, 서로 다른 물리적 서버에 존재

**특징**

- 샤드 간 독립성 → **장애 격리, 확장성 뛰어남**
- 샤딩 키(예: 사용자 ID, 지역 코드 등)를 기준으로 데이터 분산
- **라우팅 로직 필요** (어떤 샤드에 접근할지 결정해야 함)
- 예: 대규모 게임 서버, SNS 사용자 테이블

---


### 파티셔닝 (Partitioning)

- **하나의 데이터베이스 내**에서 테이블 데이터를 **논리적으로 분할**하는 방식
- 파티션은 내부적으로 나뉘지만, 외부에서는 **하나의 테이블처럼 보임**
- 주로 **성능 최적화와 관리 효율**을 위해 사용

**종류**

1. **수평 파티셔닝**: 행 기준 분할 → 날짜별, 범위별 파티션 (예: 월별 주문 테이블)
2. **수직 파티셔닝**: 열 기준 분할 → 자주 조회되는 컬럼만 분리 저장

**특징**

- **단일 DB 서버**에서 파티션별로 관리됨
- DBMS가 파티션 접근을 자동 최적화
- 샤딩과 달리 **물리적 서버 분산 아님**

---


### 비교 요약


| 항목     | 샤딩 (Sharding)     | 파티셔닝 (Partitioning) |
| ------ | ----------------- | ------------------- |
| 저장 위치  | 여러 서버(분산 DB)      | 하나의 서버(DB 내부)       |
| 분산 방식  | 물리적 분산            | 논리적 분할              |
| 확장성    | 수평 확장 (Scale-out) | 제한적 확장 (Scale-up)   |
| 관리 복잡도 | 높음 (라우팅, 중복 등)    | 낮음 (DBMS 내부 처리)     |
| 장애 격리  | 샤드 단위로 가능         | 전체 DB 영향 가능         |
| 적용 예시  | 글로벌 사용자 시스템       | 기간별 로그, 대용량 주문 테이블  |

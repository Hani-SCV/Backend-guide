### 개념


> - 여러 대의 노드에 데이터를 나누어 저장하고  
>   
> - 네트워크를 통해 동시 접근·처리 가능하도록 구성된 시스템  
>   
> - 목표: 수평 확장, 고가용성, 내구성 확보


### 분산 DB 구성 방식

1. 수평 분할 (Sharding)
    - 데이터 범위나 해시 기준으로 쪼개 여러 서버에 분산
    - 장점: 대규모 처리·부하 분산에 강함
    - 단점: 샤드 간 데이터 불균형, 라우팅 로직 복잡
2. 복제 (Replication)
    - 동일 데이터를 여러 노드에 복사 저장
    - 장점: 읽기 성능 향상, 장애 격리
    - 단점: 복제 지연(lag) 발생, 동기화 오버헤드

### 일관성 모델 종류


| 모델                        | 설명                        | 예시 시스템                       |
| ------------------------- | ------------------------- | ---------------------------- |
| 강력한 일관성                   | 모든 읽기가 가장 최신 쓰기 결과를 반영    | Google Spanner, HBase (sync) |
| 최종 일관성                    | 시간이 지나면 모든 노드가 같은 상태로 수렴  | Cassandra, DynamoDB          |
| 인과적 일관성                   | 인과 관계가 있는 연산 순서를 보장       | Riak, Cosmos DB              |
| 세션 일관성 (Read-your-writes) | 같은 세션 내 읽기는 자신의 이전 쓰기를 반영 | Cosmos DB, DynamoDB 세션 옵션    |
| 모노토닉 읽기 일관성               | 이전에 읽은 값보다 과거로 돌아가지 않음    | MongoDB 리플리카셋                |


### 모델 선택 가이드

- 금융·결제처럼 정확성 필수 → 강력한 일관성
- 대용량 로그, 피드 서비스처럼 응답 속도 우선 → 최종 일관성
- 사용자 세션 유지 → 세션 일관성
- 복잡한 이벤트 흐름 → 인과적 일관성

### CAP 이론과 PACELC

- CAP 이론: 분산 시스템은 일관성(C)·가용성(A)·파티션 허용(P) 중 두 가지만 최적화
    - CP, AP 조합으로 설계
- PACELC 확장:
    - P(Partition) 발생 시 A vs C
    - Else 정상 상태에서는 L(Latency) vs C 트레이드오프

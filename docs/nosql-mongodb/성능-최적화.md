> 💡 MongoDB는 대규모 데이터를 처리하는 데 강력한 성능을 제공하지만, 데이터가 많아질수록 최적화가 중요해집니다. 이 챕터에서는 MongoDB 성능을 최적화할 수 있는 다양한 방법을 살펴봅니다.


---


### 인덱스 최적화


인덱스는 쿼리 성능을 크게 향상시킬 수 있지만, 과도한 인덱스 사용은 성능을 저하시킬 수 있습니다. 불필요한 인덱스를 제거하고 적절한 인덱스를 사용하여 성능을 최적화합니다.


### **인덱스 분석**

- `explain()` 명령어를 사용하여 쿼리 실행 계획을 확인할 수 있습니다.

**예시: 쿼리 성능 분석**


```shell
db.myCollection.find({ age: { $gt: 25 } }).explain("executionStats")
```


### **인덱스 유지 관리**

- 주기적으로 인덱스를 재빌드하여 성능을 유지합니다.

---


### 데이터 정리


**불필요한 데이터**를 삭제하거나 **데이터를 압축**하여 데이터베이스 크기를 줄이는 것도 성능 향상에 중요합니다. 이를 통해 디스크 공간을 절약하고 읽기 성능을 개선할 수 있습니다.


### **데이터 정리 방법**

- `drop()` 명령어로 불필요한 컬렉션을 삭제하고, **TTL(Time to Live)** 인덱스를 사용해 만료된 데이터를 자동으로 삭제할 수 있습니다.

**예시: TTL 인덱스 생성**


```shell
db.sessions.createIndex({ "lastAccessed": 1 }, { expireAfterSeconds: 3600 })
```


---


### 쿼리 최적화


MongoDB에서 **복잡한 쿼리**는 성능을 크게 저하시킬 수 있습니다. 쿼리 최적화를 위해 **효율적인 필터링**과 **프로젝션**을 활용합니다.


### **쿼리 필터링 최적화**

- 쿼리에서 **필요한 데이터만** 요청하고, 불필요한 데이터를 제외합니다.

**예시: 필요한 필드만 선택**


```shell
db.users.find({}, { name: 1, age: 1 })
```


### **프로젝션 최적화**

- **$project** 연산자를 사용하여 필요한 데이터만 반환하도록 쿼리를 최적화합니다.

**예시: 특정 필드만 반환**


```shell
db.users.aggregate([
  { $project: { name: 1, age: 1 } }
])
```

### 샤딩 (Sharding)


MongoDB는 **샤딩**을 사용하여 대규모 데이터를 분산 저장하고, 성능을 최적화할 수 있습니다. 샤딩은 데이터를 여러 서버에 분배하여 **수평 확장**을 가능하게 합니다.


### **샤딩 설정**

- **샤딩 키**를 사용하여 데이터를 여러 서버에 분배합니다.

**예시: 샤딩 설정**


```shell
sh.shardCollection("mydb.users", { "user_id": 1 })
```


---


### 캐싱


**캐시**는 데이터베이스 성능을 최적화하는 중요한 방법 중 하나입니다. 자주 조회되는 데이터를 **메모리 캐시**에 저장하여 빠른 응답을 제공합니다.


### **Redis와 같은 외부 캐시 시스템 사용**

- 자주 조회되는 데이터를 Redis와 같은 캐시 시스템에 저장하여 성능을 향상시킬 수 있습니다.

### 개념


> - 그래프 데이터 모델 기반의 **그래프 데이터베이스**  
>   
> - 노드(Node)와 관계(Relationship)로 데이터 표현  
>   
> - 복잡한 연결 데이터 탐색과 분석에 최적화


### 특징

- ACID 트랜잭션 지원
- Cypher 쿼리 언어 사용 (SQL과 유사하나 그래프 특화)
- 실시간 쿼리 성능 우수
- 분산 및 클러스터링 지원 (Enterprise Edition)
- 다양한 언어용 드라이버 지원 (Java, Python, JS 등)

### 장점

- 복잡한 네트워크, 추천 시스템, 소셜 그래프에 강점
- 직관적 데이터 모델링과 쿼리 가능
- 스키마 유연성 높음
- 뛰어난 성능과 확장성

### 단점

- 관계가 적은 단순 데이터에는 부적합
- 초기 학습 곡선 존재
- 엔터프라이즈 기능 유료

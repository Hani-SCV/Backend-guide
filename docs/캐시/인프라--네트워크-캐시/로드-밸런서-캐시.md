### 🔍 개념


로드 밸런서는 여러 서버에 요청을 분산하는 네트워크 장치 또는 소프트웨어다.


**로드 밸런서 캐시**는 이 계층에서 응답을 미리 저장해두고,


일정 요청에 대해 **백엔드 서버 접근 없이 응답**하는 구조를 의미한다.


대개 **L7 로드 밸런서**(HTTP 기준)에서 적용된다.


대표 도구: **AWS ELB + CloudFront**, **Nginx**, **HAProxy**, **Envoy**


---


### ⚙️ 동작 흐름

1. 클라이언트 → 로드 밸런서 요청
2. 캐시에 응답 존재 시 → 캐시 응답 전달 (Cache Hit)
3. 없으면 → 백엔드 서버로 전달 → 응답 캐싱 (Cache Miss)
4. 동일 요청 시 캐시 응답 반복 활용

---


### 🧩 캐시 적용 예시

- 정적 HTML, 이미지, JS/CSS
- 특정 API 응답(JSON)
- 서버 상태값, 구성 정보 등

---


### ✅ 장점

- 백엔드 부하 감소
- 응답 속도 향상
- 로드 밸런서 하나로 캐싱/분산 처리 통합

---


### ⚠️ 단점 및 주의사항

- 사용자 맞춤형 콘텐츠 캐싱 주의
- 캐시 무효화가 어렵거나 느릴 수 있음
- 캐시 정책 미흡 시 오래된 응답 노출 가능

---


### 🔒 보안 고려사항

- 인증/권한 정보 포함 응답 캐싱 금지
- `Vary`, `Cache-Control` 설정 정확히 지정
- HTTPS 트래픽 중개 시 인증서 및 키 관리 필요

---


### 💡 캐시 제어 전략

- `Cache-Control: public, s-maxage=300`
- `Vary: Accept-Encoding, Authorization`
- 로그인 필요 API → `Cache-Control: private, no-store`

### 🔍 개념


CDN(Content Delivery Network)은 사용자와 가까운 위치의 서버(엣지 서버)를 통해 정적 콘텐츠를 빠르게 제공하는 분산 네트워크다. 이미지, JS, CSS, 영상 등의 정적 자산을 캐싱하여 원서버 부하를 줄이고, 지연 시간을 단축시킨다.


---


### ⚙️ 동작 흐름

1. 클라이언트가 웹 리소스를 요청 (예: 이미지)
2. CDN 엣지 서버에 해당 리소스가 있는 경우 → 바로 응답 (Cache Hit)
3. 없을 경우 → 원서버에 요청 후 받아온 리소스를 엣지 서버에 저장 (Cache Miss)
4. 이후 동일 요청은 CDN이 직접 응답

---


### 🧩 캐시 구성 요소

- **TTL (Time To Live)**

    리소스가 CDN 서버에 저장되는 시간. 보통 Cache-Control 헤더 기반으로 설정됨.

- **캐시 키(Cache Key)**

    리소스를 구분하는 기준. URL 외에도 쿼리 파라미터, 헤더 등을 포함할 수 있음.

- **Origin Server**

    실제 콘텐츠가 존재하는 원본 서버

- **Invalidation / Purge**

    캐시 무효화 또는 강제 삭제. 업데이트가 필요한 경우 수동 처리하거나 API 호출로 제어함.


---


### ✅ 장점

- 정적 리소스 로딩 속도 향상
- 트래픽 분산 → 원서버 부하 감소
- 글로벌 사용자 대상 빠른 응답 보장
- 장애 시 백업 서버 역할 가능

---


### ⚠️ 단점 및 주의사항

- **실시간 데이터엔 부적합**

    캐시된 데이터가 최신이 아닐 수 있음

- **캐시 무효화 복잡성**

    버전 관리나 Invalidation 로직이 필요함

- **비용 고려**

    고성능 CDN은 유료 요금제가 많음


---


### 🔒 보안 고려사항

- HTTPS 적용된 CDN 사용
- 오리진 서버 접근 제한 (CDN에서만 접근 허용)
- 캐시 가능한 민감 정보 차단 (Cache-Control 설정)

---


### 💡 캐시 제어 전략

- `Cache-Control: public, max-age=31536000`

    → 브라우저와 CDN 모두 1년 캐시

- 리소스 파일에 해시 붙이기 (예: `app.abc123.js`)

    → 캐시 무효화 없이 변경 가능

- `Vary` 헤더 활용 (예: `Vary: Accept-Encoding`)

    → 사용자 환경별로 다른 캐시 구성 가능

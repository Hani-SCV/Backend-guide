### 🔍 개념

- 웹 스토리지는 클라이언트 브라우저 내에 데이터를 저장하는 API
- **LocalStorage**: 도메인별 영구 저장, 세션 종료 후에도 유지
- **SessionStorage**: 탭 단위 저장, 탭 종료 시 데이터 삭제
- 둘 다 키-값 방식, 동기 처리 방식이며, 쿠키보다 큰 저장 공간 제공

---


### ⚙️ 필수 공부 항목

- **LocalStorage와 SessionStorage 차이 및 사용 시점**
- 저장 용량 제한 (대략 5~10MB, 브라우저별 상이)
- 데이터 저장/조회 API (`setItem`, `getItem`, `removeItem`, `clear`)
- 동기 방식 특성에 따른 성능 영향
- JSON 직렬화/역직렬화 처리법 (객체 저장 시)
- 저장 공간 한계와 관리 전략

---


### ✅ 실무 적용 팁

- UI 상태, 폼 입력값 등 비민감 데이터 임시 저장에 활용
- 서버 요청 최소화를 위한 캐싱 보조 수단으로 사용 가능
- 로그인 토큰 직접 저장 지양 (보안 취약)
- SPA(싱글 페이지 애플리케이션)에서 상태 유지 용이

---


### 🔒 보안 및 주의사항

- **XSS 공격에 노출되기 쉬움** → 민감정보 절대 저장 금지
- CSP(Content Security Policy) 적용으로 스크립트 통제 필수
- HTTPS 사용 및 도메인 정책 준수
- 세션 스토리지 특성상 탭별 데이터 분리로 충돌 방지 가능
- 쿠키와 달리 HTTP 요청에 자동 전송 안 됨 → CSRF 위험 적음

---


### 🛠 확장 학습 주제

- IndexedDB와 비교 (대용량 비정형 데이터 저장)
- 서비스 워커와 연계한 오프라인 캐시 전략
- 웹 스토리지 이벤트(`storage` 이벤트) 활용법
- 프론트엔드 상태 관리 라이브러리(Redux 등)와의 관계

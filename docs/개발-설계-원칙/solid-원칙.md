### 개념


> - 객체지향 설계의 다섯 가지 핵심 원칙을 뜻함  
>   
> - 코드의 **유지보수성, 확장성, 재사용성**을 높이기 위해 고안됨  
>   
> - Robert C. Martin(“Uncle Bob”)이 정리


### 각 원칙

1. **S - 단일 책임 원칙 (Single Responsibility Principle, SRP)**
    - 한 클래스는 하나의 책임만 가져야 한다
    - 변경 이유도 하나여야 함
2. **O - 개방-폐쇄 원칙 (Open/Closed Principle, OCP)**
    - 확장에는 열려있고, 변경에는 닫혀 있어야 한다
    - 기능 추가는 클래스 변경 없이 가능해야 함
3. **L - 리스코프 치환 원칙 (Liskov Substitution Principle, LSP)**
    - 자식 클래스는 부모 클래스의 역할을 대체할 수 있어야 한다
    - 자식 클래스는 부모가 기대하는 행위를 반드시 지켜야 함
4. **I - 인터페이스 분리 원칙 (Interface Segregation Principle, ISP)**
    - 클라이언트는 자신이 사용하지 않는 인터페이스에 의존하면 안 된다
    - 여러 작은 인터페이스가 낫다
5. **D - 의존 역전 원칙 (Dependency Inversion Principle, DIP)**
    - 고수준 모듈은 저수준 모듈에 의존하면 안 되고,
    - 둘 다 추상화에 의존해야 한다
    - 구체적인 것에 의존하지 말고, 추상화에 의존하라

### 요약


| 원칙  | 핵심 내용             |
| --- | ----------------- |
| SRP | 한 클래스, 한 책임       |
| OCP | 변경하지 말고 확장하라      |
| LSP | 자식은 부모 역할 대체 가능해야 |
| ISP | 필요한 인터페이스만 제공     |
| DIP | 추상화에 의존하라         |


### 효과

- 코드가 견고해지고 변경에 유연해짐
- 유지보수가 쉽고, 테스트하기 편해짐
- 설계가 명확해져 협업도 수월해짐

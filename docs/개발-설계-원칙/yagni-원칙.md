### 🧠 개념

- *“당장 필요하지 않은 기능은 만들지 마라”**는 개발 원칙
- 과도한 기능 구현으로 인한 복잡성 증가와 낭비 방지 목적

---


### ✍️ 핵심 메시지

- 미래에 필요할 것 같아도 지금은 만들지 말고,
- 실제 요구가 있을 때 구현하라
- **단순함과 효율성을 유지하는 게 중요**

---


### ✅ 적용법

- 요구사항이 명확할 때만 기능 추가
- 미리 복잡한 확장 구조 설계하지 않기
- 기능 추가는 필요 시점에 맞춰 점진적으로 진행
- 테스트 코드도 불필요한 부분 작성 금지

---


### ⚠️ 주의점

- 너무 엄격하게 적용하면, 나중에 확장 어려워질 수 있음
- 기본 설계에서 어느 정도 확장성 고려는 필요
